# Reference workflow records

Each workflow record must identify:

- the capability and evidence source;
- the dataset artifact and checksum;
- the exact legacy executable and invocation;
- relevant UI state, including component, level, slice, palette, and range;
- numerical outputs and screenshots;
- expected differences permitted in the modern application;
- the person or automated capture responsible for the record.

Do not store references to mutable external paths as the only copy of an input.
